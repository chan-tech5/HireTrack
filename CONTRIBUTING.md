# Contributing to HireTrack

Thank you for your interest in contributing to HireTrack! As an open-source project, we welcome contributions of all kinds, including bug fixes, new features, documentation improvements, and feedback.

## How to Contribute

### 1. Reporting Bugs & Feature Requests
*   Search the existing Issues to see if your bug or request has already been reported.
*   If not, open a new Issue, providing clear steps to reproduce the bug or details about the feature proposal.

### 2. Submitting Pull Requests (PRs)
*   Fork the repository and create a new branch from `master` (e.g. `feat/my-feature` or `fix/bug-desc`).
*   Ensure the project builds successfully by running `npm run build` and has no typecheck errors.
*   Write clear, concise commit messages.
*   Open a Pull Request describing your changes and link it to any relevant issues.

## Local Development Guidelines
*   Keep dependencies up to date where appropriate.
*   Follow clean code architecture principles (Server Actions separated from reusable UI elements).
*   Format code with Prettier and ensure strict TypeScript configurations are met.
